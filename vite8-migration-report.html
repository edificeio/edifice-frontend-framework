Rapport préliminaire

Migration Vite 5 → Vite 8

Audit du monorepo edifice-frontend-framework en vue de la montée de version vers Vite 8. Périmètre strict : ne toucher que ce qui est nécessaire au passage de version, aucune autre amélioration.

8 fichiers vite.config 2 apps + 6 packages Vitest 2.1.9 Vite 5.4.14 → 8.0 (12 mars 2026) Rolldown + Oxc remplacent Rollup + esbuild Prérequis : Storybook 10 mergé d'abord Stratégie : paliers 5 → 6 → 7 → 8

0 Contexte de la migration

Prérequis : ordre de merge

Storybook 9 ne supporte pas Vite 8 (peer dep plafonnée à Vite 7). Le support arrive en Storybook 10.2.19 (PR #33788). La branche dédiée Storybook 10 doit donc être mergée avant cette PR Vite 8, en visant ≥ 10.2.19 (idéalement la dernière 10.4.x).

Changement d'architecture Vite 8

Étage Vite 5/6/7 Vite 8
Bundler (prod + dev)Rollup + esbuildRolldown
Transformer JS/TS/JSXesbuildOxc
Minification CSSesbuildLightning CSS
Couche de compat : Rolldown conserve l'API plugin Rollup ; les options dépréciées (esbuild, esbuildOptions, rollupOptions) sont auto-converties.
Limite Oxc : le property mangling n'est pas supporté. Impact ciblé sur minifyIdentifiers: false utilisé chez nous → cf. 2-α.
Exception au périmètre strict : remplacement de rollup-plugin-pure et rollup-plugin-visualizer par leurs équivalents natifs Rolldown → cf. 2bis.

2 Étapes de migration (par paliers)

Branche dédiée, commits atomiques. Un palier = un commit (5 → 6, 6 → 7, 7 → 8) pour faciliter la traçabilité et le bisect en cas de régression.

  • Préparer la branche et figer la baseline
    Confirmer Storybook ≥ 10.2.19 sur develop. Brancher, lancer pnpm install && pnpm build && pnpm test && pnpm docs:build + build tiptap-playground pour capturer la baseline (taille bundles, snapshots Chromatic).
  • Mettre à jour les engines Node + CI
    "node": "^20.19.0 || >=22.12.0" dans /package.json, aligner GitHub Actions et image Docker éventuelle.
  • Palier 1 : Vite 5 → 6 (commit dédié)
    Bumper vite + plugins compatibles + Vitest 3. Lire la migration guide v6 (Sass legacy API supprimée — non utilisée chez nous). Valider build + tests sur tout le monorepo.
  • Palier 2 : Vite 6 → 7 (commit dédié)
    Bumper vite + plugins. Migration guide v7 : build.target par défaut bascule sur baseline-widely-available, split-vendor-chunk-plugin retiré. Valider.
  • Palier 3 : Vite 7 → 8 (le gros morceau)
    Bumper vite, @vitejs/plugin-react, vite-plugin-dts, vite-tsconfig-paths dans le catalog. Bumper Vitest 4. Renommer build.rollupOptions → build.rolldownOptions (6 fichiers) et optimizeDeps.esbuildOptions → optimizeDeps.rolldownOptions (Storybook). Appliquer sections 2-α (mangle) et 2bis (remplacement plugins).
  • Auditer le plugin custom remove-display-name
    Confirmer qu'il n'utilise aucun des hooks Rolldown supprimés (shouldTransformCachedModule, resolveImportMeta, renderDynamicImport, resolveFileUrl) et déclare moduleType: 'js' si nécessaire.
  • Régénérer le worker MSW (si msw bumpé)
    pnpm exec msw init apps/docs/public --no-save.
  • Passer la checklist post-migration (section ✓)
    Tous les points doivent être verts avant d'ouvrir la PR.

2-α Réécriture minifyIdentifiers → mangle

Réécriture explicite (Option A) — l'auto-conversion ne documente pas cette sous-option. minifyIdentifiers (variables/fonctions locales) a un équivalent Oxc mangle ; à ne pas confondre avec mangleProps (propriétés d'objet, non supporté par Oxc et non utilisé chez nous). 

// packages/react/vite.config.ts ET packages/extensions/vite.config.ts

// retirer :
esbuild: { minifyIdentifiers: false },

// ajouter :
build: {
  rolldownOptions: {
    output: { minify: { mangle: false } },
  },
},
🔬 Investigation à reporter en follow-up (hors PR) : minifyIdentifiers: false n'est pas justifié dans le code. Lancer git log -p sur l'apparition de cette option pour décider si on peut repasser à mangle: true et économiser de la taille de bundle. Cette PR de migration préserve le comportement à l'identique.

2bis Remplacement des plugins Rollup par les natifs Rolldown

Exception au périmètre strict actée. @babel/plugin-transform-react-pure-annotations reste utile (indépendant) et n'est pas retiré.

rollup-plugin-pure → treeshake.manualPureFunctions

// packages/react/vite.config.ts
// retirer : import { PluginPure } from 'rollup-plugin-pure';
// retirer : PluginPure({ functions: ['Object.assign'] }) du tableau plugins

build: {
  rolldownOptions: {
    treeshake: {
      manualPureFunctions: ['Object.assign'],
    },
  },
},

3 Points d'attention résiduels

Points non couverts ailleurs dans le rapport. Le reste (Rolldown, Oxc, Node, plugins, mangle…) est détaillé dans les sections 0a / 0b / 2-α / 2bis.

🧹 CommonJS interop

Vite 8 normalise le default import depuis les modules CJS : résout vers module.exports sauf si __esModule est true. Peut affecter tiptap-playground (libs CJS) et commonjsOptions.transformMixedEsModules.

🌐 Proxy regex + loadEnv

tiptap-playground/vite.config.ts utilise des clés de proxy en regex et loadEnv. Tester explicitement après bump.

🧷 Resolutions / overrides

Conserver resolutions.katex et pnpm.overrides["@rushstack/terminal"]. Vérifier qu'ils restent satisfaits par le nouvel arbre.

📐 tsconfig paths natif (info)

Vite 8 a un support natif. vite-tsconfig-paths devient remplaçable — hors périmètre de cette PR.
